# 选择性帧拆分策略

默认不拆几十万帧。帧拆分是标注之后的例外处理，不是标注工程的必经步骤。

## 先尝试现代重建

以下对象优先使用已有图片、SVG、少量 atlas、Phaser tween、粒子、混合模式、拖尾和镜头反馈重建：

- 简单斩击、光效、弹体和命中闪光。
- 可由少量贴图加参数复现的技能特效。
- UI 面板、按钮、图标等静态或低帧率资源。
- 可直接使用图片或分层 tile 还原的地图背景。

AS3 用来校准生成时机、方向、跟随对象、持续帧数、命中窗口和伤害段数，不作为现代资产组织模板。

## 进入拆分评估的条件

只有满足下列情形之一，才允许把标注状态候选改为 `needs-splitting`：

- 角色本体姿态和帧间变化无法靠现有素材或少量补绘接近。
- MovieClip 含复杂嵌套动画、mask、filter、morph shape 或逐帧手绘变化。
- 特效识别度很高，参数化重建后明显不像原版。
- 地图或 UI 依赖时间轴中的精确层级、遮罩或关键帧。
- 用户明确要求某个对象达到逐帧近似标准。

评估对象必须是单个符号或动作族，例如 `Role1Bullet1/3/4/5`，不能是整个 EXE、整个 SWF 或“全部角色”。

## 判定门

| 问题 | 通过条件 |
| --- | --- |
| 是否有明确来源链？ | 有 AS3 名称、SymbolClass 或可追踪 id。 |
| 是否影响当前切片？ | 影响正在复现的角色、怪物、关卡或 UI。 |
| 是否已有轻量方案？ | 已描述 Phaser/少量贴图方案及其具体差距。 |
| 差距是否影响感知？ | 动作读感、技能识别或场景辨识明显受损。 |
| 是否能窄拆分？ | 能限定到单个符号、动作或短帧段。 |
| 是否可验收？ | 有截图、录屏、帧数证据或明确人工标准。 |

前两项、窄拆分和可验收必须通过；其余项如未通过，默认继续使用占位或轻量重建。

## Agent 与人工边界

Agent 可以：

- 整理符号、帧数、时间轴和现有导出证据。
- 比较参数化方案与帧动画方案的成本和缺口。
- 给出 FFDec 中需要定位的精确 SWF、符号和动作范围。
- 在 FFDec CLI 已安装且用户明确授权时，从 `local-resources/regima/source/restored-swfs/` 执行只读检查或选择性导出，并把结果写入新的 `local-resources/regima/task-outputs/<batch>/`。
- 接收用户导出的最小文件族后继续整理 atlas/manifest 建议。

人工需要：

- 提供原版画面或判断视觉差距是否值得成本。
- 当 FFDec 尚未安装、CLI 无法完成或必须通过 GUI 视觉选择时，手工安装/操作工具。
- 只导出 Agent 指定的单个符号或动作族，并把结果放在新目录；不要覆盖 `local-resources/regima/legacy-extraction/`。

## 从轻到重的方案

1. 选择性导出单个符号或动作族为 PNG 序列，再打 atlas。
2. 从原版运行画面截取少量关键帧，用于临摹或对齐。
3. 复用已导出的 SVG shape，手工组合为现代图层。
4. 使用少量关键帧 atlas，加 Phaser tween 补过渡。
5. 只对高识别度部分使用帧动画，其余仍用粒子和 tween。

## 最小交付

一次真正拆分最多交付一个资源族：

- 一组 PNG 序列或 atlas。
- 来源 SWF、符号、动作、帧数和帧率说明。
- stableKey 与 manifest 注册建议。
- 在现代场景中的观察方法和剩余差异。

拆分完成不等于自动接入代码。接入应作为后续窄游戏 task 执行。

仍然禁止全量导出几十万帧、把不完整派生帧目录作为主资料来源，或因单个资源问题重启整个资源获取工程。
