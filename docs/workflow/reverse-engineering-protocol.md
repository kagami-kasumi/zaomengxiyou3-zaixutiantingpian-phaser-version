# 玩法逆向证据协议

本文规定从 Flash/AS3、恢复 SWF 和运行时观察建立现代实现输入时，证据必须如何追踪、分级、交接和验收。目标不是逐行翻译 AS3，而是可靠恢复玩家能够观察到的外观、玩法、数值、手感和流程，禁止用“看起来合理”的现代逻辑填补尚未查清的原版行为。

## 适用范围

- `TASK-SETTINGS-*` 的行为、关卡、机制或资源逆向。
- 任何准备依据 AS3、时间轴、SWF 几何或旧提取结果修改 `src/` 的任务。
- 对 `mechanics-index.md` 标记“已扒”，或对逆向文档使用“权威实现输入”“已闭合”等结论。
- 用户运行反馈与既有逆向结论冲突时的重新调查。

纯现代工程重构不需要重新逆向，但只要它改变可观察行为，就必须回到对应行为合同核对。

## 六段证据链

逆向结论按以下顺序建立。某一段与目标行为无关时可以标记“不适用”，但必须说明理由，不能静默跳过。

1. **关卡/对象局部证据**：目标关卡类、对象类、实例属性、时间轴脚本、刷怪点和停点等局部事实。
2. **共享运行时调用链**：输入、角色基类、物理世界、镜头、状态机、存档和通用结果流程等实际消费局部数据的共享系统。不得因找到关卡类就停止追踪。
3. **SWF 几何与坐标语义**：恢复源 SWF、SymbolClass、MovieClip 时间轴、嵌套矩阵、注册点、碰撞盒、显示边界，以及 local/world/screen 坐标转换。AS3 中的 `x/y` 不自动等于现代图片的左上角、中心或脚底。
4. **可观察行为合同**：把证据转换为与架构无关的输入、状态、触发条件、坐标空间、边界、时序和玩家可见结果。
5. **现代实现映射**：说明哪个现代 owner 消费该合同；AS3 类结构不得直接决定 TypeScript 架构。无法由证据确定的现代设计选择必须单独标记。
6. **双重验证**：确定性测试验证合同，运行时人工/浏览器观察验证视觉、持键、镜头、时序和组合路径。只满足其中一项不得宣称完整复现。

## 证据分级

每条关键结论必须使用以下一种标签：

- `确认事实`：存在可定位的一手证据，且证据直接支持该结论。
- `交叉确认`：至少两类独立证据一致，例如 AS3 调用链与 SWF 时间轴/运行态一致。
- `推断`：证据不完整但存在可解释推导，必须记录推导过程和反证条件。
- `未知`：当前证据不能回答；必须生成后续搜索或运行校验项。
- `现代设计选择`：原版证据不决定的现代架构、占位表现或兼容策略，不得伪装成原版事实。

不得把 `推断`、`未知` 或 `现代设计选择` 写成“原版就是如此”。用户反馈与结论冲突时，原结论立即降级为 `待复核`，直到重新闭合证据链。

## 必须留下的逆向产物

逆向笔记可以写入对应的 `docs/reverse-engineering/*-index.md`，但必须包含以下最小证据矩阵：

| 行为合同项 | 局部证据 | 共享调用链 | 几何/坐标证据 | 证据等级 | 未知与反证条件 | 验证方式 |
| --- | --- | --- | --- | --- | --- | --- |
| 示例 | 精确文件、类、行号/实例 | 实际消费者与调用路径 | 坐标空间、注册点、矩阵或不适用理由 | 确认事实/交叉确认/推断/未知/现代设计选择 | 尚未确认内容 | 自动测试 + 运行观察 |

涉及位置、碰撞、镜头或视觉对齐时，还必须明确记录：

- 源坐标空间和目标坐标空间；
- 显示对象注册点、碰撞盒、可见边界和缩放/嵌套矩阵；
- 数值推导公式及每个输入的来源；
- 现代素材原点与原 MovieClip 注册点如何映射。

只有聊天中的描述、没有文件落点和证据引用，不算可交接逆向产物。

## UI 显示列表与视觉基准门禁

涉及菜单、HUD、功能页、弹窗或任何原版 UI 时，六段证据链还必须建立可交接的 **显示列表清单**。整页 PNG/SVG、根 MovieClip 尺寸或少量 AS3 `x/y` 不能代替显示列表。清单至少逐项记录：

- 根 SymbolClass / MovieClip / character id、目标帧和舞台尺寸；
- 每个可见 child 的实例名、character id、depth、父子关系、注册点、局部矩阵、嵌套后的舞台坐标和可见边界；
- mask、filter、blend mode、alpha、scale、rotation、九宫格或裁切；
- 静态文字、动态 TextField、字体/字号/颜色/描边/滤镜，以及数据写入来源；
- SimpleButton / DefineButton2 的 up/over/down/hittest/selected、状态内矩阵和命中区；
- AS3 动态 `addChild/addChildAt/removeChild` 创建的列表、物品格、图标、分页、角色入口和返回控件；
- 现代资源与每个原显示对象的一一映射；无法复用时标记 `未知` 或 `现代设计选择`。

UI 逆向文档还必须保存或精确引用 **原版视觉基准**：来自可追溯 Flash 运行态、原 SWF 渲染或用户提供的原版截图/录屏，并记录版本、入口、状态、舞台尺寸和裁切方式。若当前无法运行 Flash，必须明确写为缺口，不能用现代页面截图反向充当原版基准。

视觉验证至少包含：

1. 相同舞台尺寸下的原版与现代版并排图或半透明叠图；
2. 对稳定区域执行像素差异、边缘差异或等价的可复查几何比较，并解释字体栅格化、抗锯齿等容差；
3. normal/hover/pressed/selected、分页、动态内容、P1/P2 和进入/退出等适用状态；
4. 一份可见对象差异清单，逐项标记“原资源复用 / 等价重建 / 用户批准的现代例外 / 未完成”。

没有显示列表清单、原版视觉基准或逐状态差异记录时，只能宣称“业务可用”“资源已接入”或“行为已复现”，不得宣称“真 UI”“UI 原生化”“视觉已复现”或“外观贴近原版”。

## 上下文与交接规则

- 逆向开始前先列“待证明的可观察问题”，不得先写现代实现再反找证据。
- 从局部类沿真实读写路径向外追踪共享系统；找到一个相关 AS3 文件不等于调用链闭合。
- 大型 AS3 和逆向文档仍按关键词窄读，但必须维护已读文件、关键命中和未追踪依赖清单。
- 发生 compact、新对话或逆向/实现人员切换后，必须从落盘证据矩阵恢复上下文；不能只依赖聊天摘要。
- 实现任务开始前必须重新读取证据矩阵引用的关键片段，确认没有 `未知` 或会影响验收的 `推断`。若存在，先补逆向，不得自行补常量或阈值。
- 现代实现发现证据冲突时停止扩写该行为，回写逆向文档并重新分级。

## 状态与关闭门禁

- `已定位`：只表示找到了候选文件/资源，不表示行为已查清。
- `已扒`：目标行为合同的局部证据、共享调用链以及所有适用的几何语义已记录；影响实现的未知项为零。
- `权威实现输入` / `已闭合`：关键合同项至少为 `确认事实`，跨坐标/视觉/时序的高风险结论必须达到 `交叉确认`；证据矩阵和反证条件齐全。
- `已复现`：在前述证据闭合基础上，现代映射、确定性测试和运行时观察全部通过。

以下情况禁止使用“已扒”“权威实现输入”“已闭合”或“已复现”：

- 只查看关卡局部类，没有追踪共享输入、物理、镜头或状态机消费者；
- 把 MovieClip 注册坐标直接当作现代图片脚底/中心，却没有矩阵和边界证据；
- 使用 AS3 中不存在、也没有运行观察支持的便利阈值或常量；
- 只有资源数量/坐标快照测试，没有持键、镜头、时序或视觉运行验收；
- 关键结论仍是 `推断` 或 `未知`，却没有在任务状态和交接中显式披露。
- UI 只加载整页背景或扁平导出物，没有追踪嵌套显示列表、动态 child、文字字段和按钮状态；
- UI 只有现代页面截图，没有原版视觉基准、逐状态对照和可见对象差异清单；
- 未经用户批准，用现代标题、面板、矩形、通用按钮或替代文字覆盖原版已有视觉。

## 逆向完成检查

1. 待证明的可观察问题是否逐项回答？
2. 是否从局部数据追到真实共享消费者？
3. 视觉/空间行为是否记录坐标空间、注册点、碰撞盒和矩阵？
4. 每条关键结论是否分级并附精确证据？
5. 是否区分原版事实、推断和现代设计选择？
6. 是否把行为写成与现代架构无关的合同？
7. 是否列出未知、反证条件和后续验证？
8. 实现前置条件和双重验证是否明确？
9. UI 是否有显示列表清单、原版视觉基准、逐状态对照和现代例外清单？

任一适用项没有答案，逆向 task 只能标为进行中、拆分或阻塞，不能归档为完成。
