# PG-005 逆向证据链不完整却宣布闭合

状态：规则已落地，运行观察中。

来源：2026-07-21 用户复盘 Stage 1-2 初版表现问题，指出只对关卡做局部逆向、忽略既有 AS3 上下文会产生幻觉，并明确要求重构逆向流程。

## 1. 问题定义

既有流程只要求“定位对应 AS3”并更新机制索引，没有强制从关卡局部数据继续追踪共享输入、角色、物理、镜头和状态机，也没有要求合并 SWF 几何、注册点、坐标空间与运行观察。证据缺环时，合理猜测可能被写成“权威输入”或“已闭合”。

这会使资源数量和局部状态测试全部通过，但玩家最基本的按键、站位、镜头和触发时机仍与原版不符；跨对话后还可能因为只保留结论、不保留证据链而重复犯错。

## 2. 证据

- Stage 1-2 初版记录只接入 P1 `A/D/J/W`，而 `KeyBoardControl.as` 已明确 P1 `[S,J,K,W]`；K 跳跃被漏掉。
- 初版把原 MovieClip 注册点 `(100,350)` 当成现代占位角色脚底，未结合 `ObsWall` 矩阵、碰撞顶面与素材原点，导致角色和怪物未站在同一地面线。
- 初版使用无直接 AS3 证据的 `stopPoint.x - 90` 世界坐标阈值；重新追踪 `ViewControllor.as` 后才发现原版在屏幕 `900..980` 区间调用 `StopPoint.touch()`。
- `agent-protocol.md` 原规则只有“先定位对应 AS3”，没有局部证据、共享调用链、SWF 几何、行为合同、现代映射和双重验证的闭合要求。

## 3. 解决方案

- 新增 `reverse-engineering-protocol.md`，建立六段证据链和统一证据分级。
- 关卡逆向必须从局部类继续追踪共享运行时消费者；视觉/空间结论必须记录坐标空间、注册点、碰撞盒、矩阵和推导公式。
- 逆向产物必须落盘证据矩阵，区分确认事实、交叉确认、推断、未知和现代设计选择。
- 实现任务必须重读矩阵引用的窄证据，不得只依赖聊天或 compact 摘要；影响验收的推断/未知未清零时先补逆向。
- “已扒、权威实现输入、已闭合、已复现”增加分级关闭门禁；自动校验确保所有入口和执行协议引用新协议。

不要求逐行翻译全部 AS3，也不要求每个任务无差别全文读取大型源文件；仍以待证明问题驱动窄查。

## 4. 测试方案

- 运行 `node --check tools/validate-workflow.mjs` 和 `npm run check:workflow`。
- 校验新协议必须包含六段证据链、五种证据等级、证据矩阵、坐标语义、上下文交接和状态关闭门禁。
- 使用内置负向样例确认删除共享运行时调用链、证据矩阵或证据分级时校验失败。
- 检查 AGENTS、CLAUDE、README、文档地图、执行协议、任务生成、代码质量门禁和逆向 researcher 均路由到新协议。
- 用 Stage 1-2 三项失败作为历史反例回放；后续至少三个逆向/实现样本按新矩阵执行。

## 5. 测试结果

- `npm run check:structure` 通过，仅保留 8 个未修改旧大文件的既有 warning。
- `node --check tools/validate-workflow.mjs` 通过。
- `npm run check:workflow` 通过；协议、任务入口、代码门禁和逆向 researcher 路由一致，任务/历史与 372 条资源标注保持有效。
- 校验器内置的三个负向样例均被拒绝：删除共享运行时调用链、删除证据矩阵或删除“推断”分级，任一情况都无法通过逆向协议契约检查。
- Stage 1-2 历史反例已按六段证据链复盘，确认三项错误分别来自共享输入遗漏、SWF 几何/注册点语义缺失和共享镜头调用链未追踪。

规则和自动门禁已落地；尚需三个后续逆向/实现样本验证流程在真实任务中持续执行，因此保持运行观察中。

## 6. 关闭标准

- 所有逆向入口都使用同一证据协议，不再只要求“找到 AS3”。
- 自动校验能拒绝缺少共享调用链、证据分级或证据矩阵的协议退化。
- 至少三个后续逆向/实现样本留下完整证据矩阵，且实现交接重新读取其关键证据。
- 用户运行反馈与逆向结论冲突时，结论能立即降级、回审并更新，而不是继续维持“已闭合”。
- 至少一个视觉/坐标样本和一个跨共享系统调用链样本通过双重验证。

仅新增协议文档、仅增加 AS3 文件列表或只跑自动测试，都只算部分缓解。
