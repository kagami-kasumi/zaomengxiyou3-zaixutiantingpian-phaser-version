# PG-006 怪物物理与死亡奖励按关卡漂移

状态：方案落地，效果观察中。

来源：2026-07-22 用户试玩 Stage 1 三关，确认 1-1 Boss 悬空且灵魂掉落只在 1-1 出现，并明确要求怪物默认遵循重力、死亡掉落由通用系统负责。

## 1. 问题定义

现代 Stage 1 的共享战斗 owner 只统一了伤害与攻击，怪物垂直物理和死亡奖励仍由各关卡 bridge 自行决定。结果是 1-1 Monster3 使用固定 y 而悬空，1-2/1-3 在创建视图时直接贴地，只有 1-1 的旧测试场景桥接调用药品、灵魂与配置化掉落。

该问题影响所有后续关卡：新增关卡若必须手工记住重力、飞行例外、死亡归属、生命/魔法/灵魂/经验结算和视图清理，规则会持续遗漏或漂移。

## 2. 证据

- 原版 `BaseObject.step()` 默认在 `!isFly` 时调用 `checkCanMove()`，只有显式飞行对象绕过地形重力；`Monster30` 明确设置 `isFly = true`、`graity = 0`，`Monster3` 没有该例外。
- `LevelSystem.activateBossArena()` 固定 `createMonster3(750, 350)`，`Monster3System` 没有 y 速度、重力或平台落地状态。
- Stage 1-2/1-3 的 view factory 各自把地面怪中心 y 直接设为地面顶减半径，证明关卡正在承担怪物物理结果。
- `TestSceneWorldBridge.spawnMonster30DropSlice()` 直接编排灵魂、药品和配置掉落；Stage 1-2/1-3 的死亡循环只删除怪物，没有奖励调用。
- 原版 `BaseMonster.dropAura()` 是共享死亡入口，统一调用 `addMedicine()`、`fallEquip()` 并创建追踪击杀归属英雄的 Aura；经验则在怪物死亡结算中直接授予，不是地面球。

## 3. 解决方案

1. 建立 `MonsterPhysicsSystem`：默认 `grounded` 物理，关卡只提供平台；只有怪物定义显式声明 `flying/hovering` 才豁免重力。
2. 建立 `MonsterDefeatRewardSystem`：接收怪物种类、死亡坐标、击杀归属、关卡上下文和可注入随机源，统一产生 `HealthPickup`、`ManaPickup`、`SoulPickup` 与直接 `ExperienceReward`。
3. 建立共享 Phaser 掉落 view adapter，三关只注册玩家快照、平台和死亡事件，不复制概率、数量、追踪或资源选择。
4. 正式资源来自恢复 `Common1.swf`/`OtherMat1.swf`；颜色和 `auraRed/auraWhile` 只属于资源映射，不作为领域类型。
5. 本轮只迁移 Stage 1 三关，不扩成完整 ECS 或全怪物 AI 重构。

## 4. 测试方案

- 纯系统测试覆盖默认重力落地、飞行豁免、平台选择和大 delta 不穿透。
- 纯系统测试覆盖每次死亡只结算一次、击杀归属、生命/魔法概率、灵魂等待/上浮/追踪、直接经验和无存活目标边界。
- Stage 1 专项测试确认三关死亡路径都调用共享 owner，且 bridge 不再定义奖励概率/数量。
- 运行 `npm run test:systems`、`npm run build`、`npm run check:workflow`、`git diff --check`；人工复验 1-1 Boss 落地及 1-2/1-3 掉落。

## 5. 测试结果

`TASK-SLICE-134` 已建立 `MonsterPhysicsSystem`、`MonsterDefeatRewardSystem` 与共享 `Stage1RewardBridge`，并迁移 Stage 1 三关：

- 默认地面怪重力和显式飞行豁免专项测试通过；1-1 Monster3 从固定悬空改为平台落地。
- 三关死亡均调用共享奖励 owner；生命/魔法概率、灵魂数量/追踪和经验配置不再定义在关卡 bridge。
- 5 个恢复 SWF 真资源已接入，资源标注 381 条校验通过。
- `test:systems`（含新专项）、`build`、`check:structure`、`check:workflow` 通过；结构检查只剩 8 个既有 warning。
- 用户运行时观感复验和后续新关卡防复发样本尚未取得，因此不提前关闭 PG-006。

## 6. 关闭标准

- Stage 1 三关均由同一怪物物理和死亡奖励 owner 驱动。
- 地面怪默认受重力，飞行例外必须来自显式怪物配置。
- 任一怪物死亡只结算一次，生命/魔法/灵魂/经验语义和归属不由关卡复制。
- 真资源、专项测试、全量测试、构建和人工复验通过。
- 至少一个后续关卡接入样本未复制上述规则后，才从“治理中”转为关闭。
